# content-review

**Purpose.** Before anything moves, understand what story the asset must tell.
Read the brief and any supplied content/data; return the **story spine**: the one
message, the ordered key beats, the audience, and the concrete data/quotes that
deserve motion.

**Output (judgment).** A structured brief the `ideate` skill consumes: message,
beats[], audience, candidate data points, tone, constraints (duration/aspect if
known).

**Drives.** Nothing deterministic — this is pure judgment.

> Status: planned (contract only in S0). See `motion/CLAUDE.md` build sequence.
